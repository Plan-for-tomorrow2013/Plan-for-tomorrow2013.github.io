JavaScript fundamentals: Some analogies

23 August 2025

JavaScript is known as the "language of the web." It is a powerful object-oriented language that is lightweight and can communicate with the browser

1. JavaScript, HTML & CSS:

HTML is the stage and the set pieces - the structure and layout. CSS is the lighting, costumes and colours - the things that make it pretty!. JavaScript is the director, instructing the actors (the page elements) when and how to move, respond, or change. It adds the interactivity that brings the performance to life.

2. Control Flow & Loops:

Imagine you're hosting a dinner party. First you plan (grocery shopping, prep dishes in order): that’s control flow, the sequence of steps you follow. But if you're prepping multiple appetizers and need to chop carrots until they’re all done—that’s a loop: repeating a task (chopping) until a condition is met (all carrots are chopped). In JavaScript, control flow ensures things execute top‑to‑bottom, while loops allow repeating actions until a condition is fulfilled.

3. What the DOM Is & How You Interact with It:

The Document Object Model (DOM) is like a tree‑structure map of your webpage—each branch and leaf represents HTML elements that JavaScript can reach and manipulate.

Here you are reaching into the DOM (document.querySelector), selecting an element by its ID ("#welcome"), and updating its content (text.content = ("Hello!")). This is how JS dynamically interacts with the page

4. Accessing Data: Arrays vs. Objects:

JavaScript structures data in different ways:

  • Arrays

    are like numbered lists (0, 1, 2…). You access items by position: myArray[0] gives the first item

  • Objects

    are like labeled drawers—you refer to values by their names: myObject.name gives you the "name" value

Arrays are great for ordered lists; objects are ideal when you need descriptive keys or named access

5. Functions: What They Are & Why They Help:

A function is like a kitchen appliance—once you define how it works (e.g., a blender: add fruits + press blend), you can reuse it whenever needed. In JavaScript, you define a function once and then call it whenever you want that set of tasks done—keeping your code clean, organized, and reusable
